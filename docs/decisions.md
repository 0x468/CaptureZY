# 已定决策

## 基础方向

- 项目名称：`CaptureZY`
- 目标平台：仅支持 Windows
- 语言标准：`C++23`
- 构建系统：`CMake`
- GUI 路线：原生 `Win32`

## 平台与功能策略

- 最低支持系统版本：`Windows 10 1903`
- 截图后端策略：优先使用 `Windows.Graphics.Capture`
- 配置存储格式：优先 `JSON`
- 发布形态：先做便携版，核心流程稳定后再补安装器
- 当前许可证：`MIT`

## 编译器策略

- 默认日常开发与发布编译器：`MSVC`
- 保留 `clang-cl` 预设，用于补充校验 warning 与编译差异

## 架构约束

不要为了未来可能切换 UI 框架而过度抽象。`core` 需要与 Windows API 解耦，但应用壳层、截图覆盖层和贴图窗口可以保持彻底原生。
