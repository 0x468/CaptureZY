# 手工回归清单

## 文档目的

这份文档用于为 CaptureZY 提供一份可以重复执行的手工回归基线。

它的目标不是覆盖所有极端场景，而是优先覆盖当前 Alpha 阶段最容易回归、最影响可用性的主路径：

- 启动与托盘
- 截图主链路
- 保存主链路
- 贴图基础交互
- 设置与默认行为
- 崩溃诊断
- DPI 与多显示器基础观察

## 建议执行时机

- 新增或修改截图主链路后
- 修改贴图窗口交互后
- 修改设置、默认行为或持久化逻辑后
- 发布候选版本前
- 修复崩溃、DPI、多显示器相关问题后

## 建议测试环境

### 最小环境

- Windows 10 或 Windows 11
- 一台主显示器
- 100% 或 125% 缩放
- `Debug` 构建一份
- `Release` 构建一份

### 推荐补充环境

- 双显示器
- 主副屏缩放不同，例如 100% + 150%
- 任务栏位于非默认位置
- 存在中文路径的保存目录

## 执行前准备

1. 关闭正在运行的 `CaptureZY.exe`。
2. 记录当前配置文件位置：
   `%APPDATA%\CaptureZY\settings.json`
3. 记录当前诊断目录：
   `%LOCALAPPDATA%\CaptureZY\Diagnostics`
4. 如果本轮需要验证崩溃诊断，建议先清空或备份诊断目录中的旧文件。
5. 确认本轮要验证的构建版本：
   - `out\build\windows-msvc\src\app\Debug\CaptureZY.exe`
   - `out\build\windows-msvc\src\app\Release\CaptureZY.exe`

## 冒烟用例

### SMK-01 启动与托盘

步骤：

1. 启动程序。
2. 确认主窗口可以显示。
3. 关闭主窗口或最小化到托盘。
4. 右键托盘图标打开菜单。

预期结果：

- 程序正常启动，没有报错弹窗。
- 托盘图标存在。
- 程序启动后默认驻留在托盘，不显示任何主窗口或状态面板。
- 托盘菜单可正常弹出。
- 菜单中可以看到 `设置...`、截图入口、贴图批量操作入口和配置入口。

### SMK-02 托盘恢复窗口

步骤：

1. 保持 `托盘单击动作` 为默认值 `打开托盘菜单`，`托盘双击动作` 为默认值 `禁用`。
2. 左键托盘图标。
3. 右键托盘图标。

预期结果：

- 左键和右键都可以正常弹出托盘菜单。
- 程序不会因为托盘交互弹出额外主窗口或状态面板。

## 截图链路

### CAP-01 区域截图并复制

步骤：

1. 在设置中将默认截图范围设为 `区域截图`。
2. 将默认截图动作设为 `仅复制`。
3. 触发截图热键。
4. 在任意支持粘贴图片的应用中粘贴。

预期结果：

- 选区覆盖层正常显示。
- 完成选区后程序回到空闲状态。
- 截图完成后程序继续驻留在托盘，不主动弹出任何主窗口或状态面板。
- 剪贴板中存在截图内容。

### CAP-01A 窗口预选截图

步骤：

1. 触发截图热键。
2. 将鼠标移动到任意普通应用窗口上。
3. 观察高亮框并单击。

预期结果：

- 覆盖层默认进入窗口预选模式。
- 鼠标悬停到窗口上时，会自动高亮整个窗口边界。
- 高亮窗口内部不再保留遮罩，选区内容会明显变亮、便于观察。
- 单击后直接截取高亮窗口。
- 截图完成后程序继续驻留在托盘，不弹出额外主窗口。

### CAP-01B 覆盖层快捷切换

步骤：

1. 触发截图热键。
2. 直接按住左键框选任意区域。
3. 再次触发截图热键并按 `Ctrl+A`。

预期结果：

- 即使覆盖层正在预选窗口，也可以随时按住左键拖拽自定义区域。
- 拖拽形成的自定义选区内部也不再保留遮罩，边框与选区内容更清晰。
- `Ctrl+A` 会直接选择整个虚拟屏幕。

### CAP-01C 覆盖层取消

步骤：

1. 触发截图热键。
2. 在覆盖层内按右键取消一次。
3. 再次触发截图热键，并按 `Esc` 取消一次。

预期结果：

- 右键和 `Esc` 都可以取消当前截图。
- 取消后程序回到空闲状态。
- 取消后程序继续驻留在托盘，不弹出额外主窗口或状态面板。

### CAP-02 区域截图并贴图

步骤：

1. 在设置中将默认截图动作设为 `复制并贴图`。
2. 触发截图热键并完成选区。

预期结果：

- 截图完成后创建新的贴图窗口。
- 程序继续驻留在托盘，不主动弹出任何主窗口或状态面板。
- 剪贴板中也保留截图内容。

### CAP-03 全屏截图并保存

步骤：

1. 在托盘菜单点击 `全屏截图并保存`。

预期结果：

- 程序直接截取整个虚拟屏幕。
- 优先保存到默认保存目录。
- 如果默认保存失败，则回退到 PNG 保存对话框。

## 保存链路

### SAVE-01 默认保存目录与前缀

步骤：

1. 打开 `设置...`。
2. 修改默认保存目录和默认文件名前缀。
3. 执行一次保存类截图。

预期结果：

- 文件保存到新目录。
- 文件名前缀符合设置值。
- `%APPDATA%\CaptureZY\settings.json` 同步更新。

### SAVE-02 打开默认保存目录

步骤：

1. 在托盘菜单点击 `打开默认保存目录`。

预期结果：

- 文件资源管理器打开当前默认保存目录。

## 贴图交互

### PIN-01 移动与缩放

步骤：

1. 创建一个贴图窗口。
2. 拖动贴图。
3. 对贴图滚轮缩放。
4. 右键贴图，观察缩放摘要并点击 `重置缩放为 100%`。

预期结果：

- 贴图可以平滑移动。
- 缩放比例变化符合预期。
- 缩放百分比浮框正常显示。
- 滚轮缩放时保持贴图左上角位置不变，不再以鼠标位置为缩放中心。
- 在相同缩放比例下重复拖动、遮挡恢复或打开菜单时，不应出现明显额外卡顿。
- 右键菜单会显示当前缩放比例摘要，例如 `缩放：125%`。
- 当缩放不为 `100%` 时，可以执行 `重置缩放为 100%` 并恢复到正常尺寸。
- 重置缩放时保留贴图当前位置，不会跳回初始截图位置。
- 将贴图拖出屏幕顶部后，松手时不应被系统自动吸附回顶部边缘。

### PIN-02 单窗口菜单操作

步骤：

1. 右键一个贴图窗口。
2. 观察菜单分组和 `始终置顶` 勾选状态。
3. 依次验证复制、保存、置顶切换、隐藏、关闭等入口。

预期结果：

- 菜单可正常弹出。
- 复制/保存、置顶状态、隐藏/关闭三组菜单项之间有分隔。
- `始终置顶` 使用固定文案，并用勾选状态表示当前是否置顶。
- 取消 `始终置顶` 后，贴图不再保持顶层；重新勾选后恢复顶层。
- 各操作与菜单文案语义一致。
- 点击“隐藏此贴图”后，该贴图从桌面消失，但不会被关闭。
- 关闭后贴图从桌面移除。

### PIN-03 批量操作

步骤：

1. 创建两个以上贴图窗口。
2. 在托盘菜单观察批量入口文案。
3. 观察批量入口上方的贴图数量摘要。
4. 依次点击批量隐藏、批量恢复、批量关闭入口。

预期结果：

- 批量操作只影响贴图，不影响主窗口与托盘。
- 批量入口上方会显示当前贴图数量摘要，例如 `贴图：显示 1 个，隐藏 2 个`。
- 菜单灰态与当前贴图状态匹配。
- 当存在隐藏贴图时，恢复入口会按数量显示，例如 `恢复 2 个隐藏贴图`。
- 当存在可见贴图时，隐藏/关闭入口会按数量显示，例如 `隐藏 3 个贴图`、`关闭 3 个贴图`。
- 当存在隐藏贴图时，关闭入口会明确提示包含隐藏贴图，例如 `关闭 3 个贴图（含 2 个隐藏）`。

### PIN-04 隐藏贴图的托盘恢复路径

步骤：

1. 创建一个或多个贴图窗口。
2. 隐藏其中至少一个贴图。
3. 打开托盘菜单，观察贴图摘要与批量恢复入口。

预期结果：

- 当存在隐藏贴图时，托盘菜单中的贴图数量摘要会体现隐藏数量。
- 恢复入口会按数量显示，例如 `恢复 1 个隐藏贴图`。
- 如果同时仍有可见贴图，摘要中也会体现仍在显示的贴图数量。

## 设置与默认行为

### SET-01 设置对话框保存

步骤：

1. 打开 `设置...`。
2. 修改默认截图范围、默认截图动作、托盘单击动作、托盘双击动作、热键、默认保存目录、默认文件名前缀。
3. 点击 `确定`。

预期结果：

- 弹出“设置已更新”。
- 配置文件同步更新。
- 托盘中的默认范围/动作勾选状态同步更新。

### SET-02 设置对话框取消

步骤：

1. 打开 `设置...`。
2. 修改若干设置但点击 `取消`。

预期结果：

- 当前运行中的设置保持不变。
- 配置文件不发生变化。

### SET-03 热键编辑期间的屏蔽

步骤：

1. 打开 `设置...`。
2. 将焦点放到热键输入控件。
3. 输入当前正在使用的全局热键组合。

预期结果：

- 不会直接触发截图。
- 热键控件正常接收组合键。
- 关闭设置对话框后，全局热键恢复工作。

### SET-04 托盘快捷项一致性

步骤：

1. 打开托盘菜单。
2. 检查设置相关入口。

预期结果：

- 保留：
  - `设置...`
  - `默认截图范围`
  - `默认截图动作`
  - `打开默认保存目录`
  - `编辑配置文件`
  - `打开配置目录`
  - `重新加载配置`
- 不再保留已被设置对话框覆盖的重复入口。

### SET-05 托盘点击动作配置

步骤：

1. 打开 `设置...`。
2. 将 `托盘单击动作` 改为 `按默认设置开始截图`，将 `托盘双击动作` 保持为 `禁用`。
3. 点击 `确定`。
4. 左键托盘图标。

预期结果：

- 左键托盘图标后，不再弹出托盘菜单。
- 程序会按当前默认截图范围和默认截图动作进入截图主链路。
- `%APPDATA%\\CaptureZY\\settings.json` 中会同步写入托盘单击/双击动作配置。

### SET-06 托盘双击动作与单击延迟

步骤：

1. 打开 `设置...`。
2. 将 `托盘单击动作` 设为 `打开托盘菜单`，将 `托盘双击动作` 设为 `按默认设置开始截图`。
3. 点击 `确定`。
4. 单击托盘图标一次。
5. 双击托盘图标一次。

预期结果：

- 启用与单击不同的双击动作后，单击会按系统双击时间略有延迟。
- 单击后会弹出托盘菜单。
- 双击后不会先弹出托盘菜单，而是直接进入截图主链路。

### SET-07 退出确认与不再提示

步骤：

1. 打开 `设置...`，确认 `退出应用前确认` 已勾选。
2. 在托盘菜单点击 `退出`。
3. 在确认对话框中点击 `取消`。
4. 再次点击托盘菜单中的 `退出`，勾选 `不再提示` 后确认退出。
5. 重新启动程序，确认 `%APPDATA%\CaptureZY\settings.json` 已写入退出确认配置。
6. 再次通过托盘菜单点击 `退出`。

预期结果：

- 第一次点击 `退出` 时会弹出退出确认对话框，取消后程序继续驻留在托盘。
- 勾选 `不再提示` 并确认后，程序正常退出。
- 重启后配置文件会保留退出确认偏好。
- 再次点击 `退出` 时，不再弹出确认对话框，程序直接退出。

## 崩溃诊断

### DIAG-01 Debug 构建的 C++ 异常自测

步骤：

1. 运行：

```powershell
& .\out\build\windows-msvc\src\app\Debug\CaptureZY.exe --diag-test-cpp-crash
```

预期结果：

- 程序弹出错误消息框。
- 诊断目录新增 `fatal-*.txt`。
- 文本中包含 `type: caught_top_level_exception`。

### DIAG-02 Debug 构建的 SEH 异常自测

步骤：

1. 运行：

```powershell
& .\out\build\windows-msvc\src\app\Debug\CaptureZY.exe --diag-test-seh-crash
```

预期结果：

- 进程异常退出。
- 诊断目录新增 `crash-*.txt` 与 `crash-*.dmp`。
- 文本中包含 `type: unhandled_exception` 和 `minidump_written: true`。

### DIAG-03 Release 构建不暴露自测开关

步骤：

1. 运行：

```powershell
& .\out\build\windows-msvc\src\app\Release\CaptureZY.exe --diag-test-cpp-crash
```

预期结果：

- 程序按正常路径启动。
- 不会因为该参数触发自测崩溃。

## DPI 与多显示器观察项

### DISP-01 高 DPI 基础观察

步骤：

1. 在 125% 或更高缩放下启动程序。
2. 执行区域截图。
3. 创建贴图并缩放。
4. 打开设置对话框和托盘菜单。

预期结果：

- 选区、贴图、设置对话框和托盘交互没有明显错位。
- 文本和控件没有严重裁切。
- 进程已按 Per-Monitor DPI 感知运行，不应出现明显的系统位图拉伸模糊。

### DISP-02 双显示器基础观察

步骤：

1. 在双显示器环境下启动程序。
2. 分别在主屏和副屏上执行区域截图。
3. 执行一次全屏截图。
4. 拖动贴图跨屏移动。

预期结果：

- 区域截图可覆盖两个屏幕中的任意位置。
- 全屏截图覆盖整个虚拟桌面。
- 贴图跨屏移动时不出现明显跳变或错误缩放。

### DISP-03 显示器热插拔观察

步骤：

1. 在程序运行期间断开或关闭副显示器。
2. 再次执行截图与贴图操作。

预期结果：

- 程序不崩溃。
- 截图与贴图基本功能仍可继续工作。
