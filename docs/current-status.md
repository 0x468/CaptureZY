# 当前阶段总览

## 文档目的

这份文档用于同步 CaptureZY 截至当前提交阶段的实际进展，并作为下一阶段规划讨论的基础草案。

它不替代 `docs/roadmap.md`，而是补充“当前已经做到什么”“还缺什么”“下一步先做什么”。

## 当前结论

- 截图主链路已经打通，不再只是原型。
- 贴图主链路已经具备基础可用性。
- 设置基础设施已经起步，Alpha 可用性收敛已进入执行阶段。
- 下一阶段重点应从“继续补单点功能”切换到“收敛交互、配置与稳定性边界”。
- 智能选区已经明确进入后续规划，但建议先从窗口级预框选开始，而不是直接进入控件级检测。

## 当前模块状态

### `core`

- 负责应用状态、状态文案与设置模型。
- 当前已包含配置对象与 JSON 持久化入口，边界仍相对清晰。

### `feature_capture`

- 已从“直接操作剪贴板的抓屏函数”演进为“截图结果对象 + 多种输出路径”。
- 当前已包含覆盖层、抓屏、窗口级预框选、保存对话框、PNG 导出和截图结果对象。

### `feature_pin`

- 已包含贴图窗口与贴图管理器。
- 当前已具备基础可用交互，但离最终产品级体验还有明显距离。

### `platform_win`

- 当前是主要调度入口。
- 负责托盘、热键、截图请求、截图结果分发和贴图批量控制。
- 主窗口当前仅作为隐藏宿主窗口存在，不再承担可见状态面板职责。
- 目前功能集中度较高，后续仍需要持续控制复杂度。

### `app`

- 负责进程入口、COM 生命周期与顶层崩溃诊断安装。
- 当前职责仍比较稳定，但已经开始承担最小运行时兜底责任。

## 已完成的关键里程碑

1. 工程骨架、格式化和静态检查配置完成。
2. Win32 应用壳层、托盘和全局热键完成。
3. 区域截图覆盖层与选区链路完成。
4. 剪贴板复制链路完成。
5. 贴图窗口原型完成，并逐步补齐移动、缩放、关闭和反馈浮层。
6. `CaptureResult` 与 `PinManager` 已引入，截图与贴图的对象边界更稳定。
7. PNG 保存流程与全屏截图命令已接入主路径。
8. 配置存储、默认截图行为、默认保存目录、热键配置与托盘配置入口已接入。

## 当前主要问题

### 交互仍然偏“功能已到位”，但还不够“产品化”

- 截图后的默认行为目前仍更像“开发期固定策略”，而不是可配置产品行为。
- 贴图的更多进阶交互仍未收敛，例如透明度、悬停入口与更细的状态反馈。
- 截图覆盖层已具备窗口级自动预框选与快捷全屏选择，但控件级预框选仍未开始。
- 截图交互设置仍较为固定，例如截图时右键取消、托盘左键动作配置和截图页设置尚未收敛。

### 持久化层和设置界面已起步，但还没有完全收敛

- 热键、默认截图动作、默认保存路径与基础命名前缀已经支持统一的图形化编辑。
- 当前保留了托盘中的默认范围/动作快速切换，完整配置则收敛到统一设置对话框。
- 贴图状态和更多产品级设置尚未持久化。

### 稳定性边界还需要专门处理

- 已具备最小崩溃日志与 dump 输出能力，但仍缺少更系统的运行日志分层。
- 已形成首版手工回归清单，但仍需要在真实高 DPI 环境持续执行。
- 已形成多显示器与热插拔基础观察项，但仍需要补充更细的回归矩阵。
- 已对当前尺寸下的贴图缩放结果增加位图缓存，但大尺寸贴图的渲染和性能仍需继续观察。
- 贴图拖拽已切换为自定义移动逻辑，以避免系统标题栏拖拽带来的顶部自动贴边行为。
- 已具备基础 ZIP 打包和安装目录导出能力，但仍缺少安装器、签名与发布自动化。
- 已显式启用进程级 Per-Monitor DPI 感知，但各窗口在不同 DPI 组合下仍需要持续回归。
- 隐藏贴图的恢复提示当前主要依赖托盘菜单语义，后续如需更强反馈可再考虑轻量提示方案。

### 代码组织还有继续收敛空间

- `platform_win/main_window.*` 仍是当前最容易继续膨胀的位置。
- 后续如果命令继续增多，需要考虑进一步拆出命令编排或应用控制层。

## 下一阶段建议目标

建议将下一阶段定义为“Alpha 可用性收敛”。

该阶段不追求大而全，而是优先让已有主路径更稳定、更一致、更像真正可日用的产品。

## 建议优先级

### 第一优先级：设置与默认行为

- 增加设置存储结构
- 支持可配置热键
- 支持默认截图动作配置
- 支持默认保存路径与基础命名规则

这是下一阶段最值得优先做的部分，因为它直接决定现有能力能否真正形成稳定工作流。

### 第二优先级：贴图交互收敛

- 统一贴图右键菜单项
- 明确复制、保存、关闭、隐藏、置顶等操作语义
- 视需要补充轻量 Hover 操作入口
- 评估透明度调节是否进入 Alpha 范围

### 第三优先级：稳定性与回归基线

- 已完成：建立首版 DPI 手工验证清单
- 已完成：建立首版多显示器验证清单
- 已完成：建立截图保存与贴图窗口基础回归清单
- 明确发布前需要人工验证的关键场景

### 第四优先级：渲染层升级

- 评估贴图窗口是否从当前 GDI 路径逐步切到更稳定的 D2D 路径
- 评估覆盖层是否需要更现代的视觉表达

这一项建议后置，因为它会带来额外技术复杂度，但对当前“可用性收敛”不是最高优先级。

### 第五优先级：智能选区增强

- 增加窗口级自动预框选
- 修正窗口预选与实际可见边界的一致性
- 增加任务栏与系统壳窗口识别
- 支持 `Ctrl+A` 直接选择全屏
- 后续再评估控件级预框选与 UI Automation 集成

这一项的产品价值很高，但实现复杂度和兼容性成本也明显更高，适合分阶段推进。

### 第六优先级：截图交互设置收敛

- 支持截图中右键取消或退出
- 支持退出确认与“不再提示”持久化
- 支持托盘左键默认动作配置
- 在设置对话框中补出独立的截图设置页

这一项更多属于产品打磨，不是底层能力突破，但对日常使用体验影响很大。

## 暂不建议优先投入的方向

- OCR
- 复杂标注工具
- 云同步
- 安装器与自动更新
- 过早做大规模渲染重构

这些方向并非不重要，而是与当前阶段的“把现有能力做稳、做顺手”相比优先级更低。

## 当前阶段的完成判定建议

当满足以下条件时，可以认为项目进入了更明确的 Alpha 阶段：

- 截图默认行为可以配置
- 基础热键可以配置
- PNG 保存路径与命名规则具备最小可用性
- 贴图窗口单体与批量操作语义清晰
- DPI、多显示器、保存与贴图流程具备基础回归清单
